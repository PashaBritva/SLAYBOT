## Что это за проект

**SLAYBOT** — Discord-бот на discord.js v14 + MongoDB (Mongoose). Автомодерация, модерация, экономика, музыка (Lavalink), гивы, тикеты, статистика, инвайты, предложения и 50+ команд.

Два инстанса бота:
- **production** (`main`) — токен `BOT_TOKEN`, PM2 процесс `slaybot`
- **debug** (`develop`) — токен `DEBUG_BOT_TOKEN`, PM2 процесс `slaybot-debug`

## Модерация — полная документация

### ModUtils — центральный класс

**Файл:** `src/helpers/ModUtils.js`

#### Проверка истории наказаний
```js
const info = await ModUtils.getPunishmentInfo(guild, target);
// Возвращает:
// {
//   warnings: 4,        // текущее кол-во предупреждений
//   maxWarn: 5,         // лимит из настроек сервера
//   warningsLeft: 1,    // сколько осталось до авто-наказания
//   hasActiveTimeout: false,
//   nextAction: "бан",  // "таймаут" | "кик" | "бан"
//   isLastWarning: true,  // это последнее предупреждение?
//   isOverLimit: false,   // уже превышен лимит?
// }
```

#### Методы действий
| Метод | Что делает | Возвращает |
|---|---|---|
| `warnTarget(issuer, target, reason)` | +1 предупреждение, проверка max_warn | `true` или код ошибки |
| `timeoutTarget(issuer, target, ms, reason)` | Discord timeout | `true` или `"ALREADY_TIMEOUT"` |
| `kickTarget(issuer, target, reason)` | Кик с сервера | `true` или код ошибки |
| `banTarget(issuer, target, reason)` | Бан с сервера | `true` или код ошибки |
| `unBanTarget(issuer, target, reason)` | Снятие бана | `true` или код ошибки |
| `vMuteTarget` / `vUnmuteTarget` | Голосовой мут/размут | `true` или `"NO_VOICE"` |
| `deafenTarget` / `unDeafenTarget` | Голосовой deafen/undeafen | `true` или `"NO_VOICE"` |
| `disconnectTarget` | Отключение от голосового | `true` или `"NO_VOICE"` |
| `moveTarget(issuer, target, reason, channel)` | Перемещение между каналами | `true` или `"ALREADY_IN_CHANNEL"` |
| `purgeMessages(issuer, channel, type, amount)` | Массовое удаление сообщений | кол-во удалённых или `"NO_MESSAGES"` |

#### Коды ошибок
`"MEMBER_PERM"` — модератор ниже цели в иерархии ролей
`"BOT_PERM"` — бот ниже цели в иерархии ролей
`"ALREADY_TIMEOUT"` — цель уже в таймауте
`"NO_TIMEOUT"` — у цели нет таймаута для снятия
`"NO_VOICE"` — цель не в голосовом канале
`"ALREADY_MUTED"` / `"NOT_MUTED"` / `"ALREADY_DEAFENED"` / `"NOT_DEAFENED"`
`"TARGET_PERM"` — цель не может подключиться к каналу
`"ALREADY_IN_CHANNEL"` — цель уже в целевом канале
`"ERROR"` — непредвиденная ошибка

#### Проверка иерархии
```js
ModUtils.canModerate(issuer, target) // true/false
// Владелец сервера может модерировать всех
// Владелец сервера не может быть модерирован никем
// Иначе: роль модератора должна быть строго выше роли цели
```

### Система предупреждений

**Логика:**
1. `/warn` → `memberDb.warnings += 1`
2. Если `warnings >= max_warn.limit` → автоматическое наказание (`max_warn.action`)
3. После авто-наказания `warnings` сбрасывается в `0`
4. `/kick` и `/ban` НЕ сбрасывают warnings автоматически (но показывают сколько было)

**Настройки сервера (Guild.max_warn):**
```js
max_warn: {
  action: { type: String, enum: ["TIMEOUT", "KICK", "BAN"], default: "KICK" },
  limit: { type: Number, default: 5 },
}
```

## CI/CD

### Пайплайны

| Файл | Триггер | Что делает |
|---|---|---|
| `.github/workflows/ci.yml` | push в main/develop, PR в main | lint → security → smoke-test → debug-test (только develop) |
| `.github/workflows/deploy.yml` | CI на main успешен | Деплой production (`slaybot`) |
| `.github/workflows/deploy-debug.yml` | CI на develop успешен | Деплой debug (`slaybot-debug`) |

### Smoke-test (main)
- 5 отдельных шагов: schemas, helpers, events, handlers, commands
- Таймауты 1-2 мин на шаг
- Без `delete require.cache` для скорости

### Debug-test (develop)
- Запускает `npm run debug:load` с `--trace-warnings`
- Полные стектрейсы при ошибках
- Таймаут 3 мин

### Локальная отладка
```bash
npm run debug:load   # Загрузить все модули с полными стектрейсами
```

### Ветвление
```
develop → CI → debug-test → deploy-debug → slaybot-debug (debug бот)
main    → CI → smoke-test → deploy        → slaybot (production)
```

## Правила кода

1. Prettier + ESLint — без исключений
2. Русский язык в пользовательских сообщениях, английский в коде/логах
3. Не хардкодить цвета — брать из конфига
4. Mongoose-модели — только через `mongoose.models[name] ? ... : ...`
5. Новые команды — в соответствующую папку `src/commands/<category>/`
6. Коммиты: conventional commits (`feat:`, `fix:`, `style:`, `ci:`, `docs:`, `chore:`)
7. Файл `.ai-context.md` НЕ коммитить — он в `.gitignore`
8. Ответ модерационных команд всегда включает статистику предупреждений (через `getPunishmentInfo`)
